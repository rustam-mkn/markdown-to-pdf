ПРОЕКТ: Генерация PDF-карточек из Markdown

═══════════════════════════════════════════════════════════════

ЦЕЛЬ ПРОЕКТА:
Создание PDF-файлов с карточками из текстового контента в Markdown.

ПРОЦЕСС (2 этапа):
1. Markdown файлы (.md) - исходный текст для карточек
2. HTML файлы (*_cards.html) - верстка карточек с стилями
3. PDF файлы (*.pdf) - векторный документ со всеми карточками

⚠️ ВАЖНО: Текст из Markdown-файлов НЕЛЬЗЯ изменять или корректировать!
   Используется только тот текст, который есть в исходных .md файлах.
   Никаких дополнений, исправлений или придумывания контента.

═══════════════════════════════════════════════════════════════

СТРУКТУРА ФАЙЛОВ:

source/
  markdown/     - Исходные Markdown файлы с текстом
  html/         - HTML файлы с карточками и template.html

scripts/
  create-pdf.js     - Генерация векторного PDF из HTML (Puppeteer + pdf-lib)
  create-all-pdfs.js - Генерация всех векторных PDF файлов

dist/           - Финальные векторные PDF файлы (готовые к использованию)

═══════════════════════════════════════════════════════════════

КАК ЗАПУСТИТЬ:

1. Создание одного PDF (Модуль 5):
   node scripts/create-pdf.js
   
2. Создание всех PDF файлов:
   node scripts/create-all-pdfs.js

Результат: Векторные PDF файлы появятся в папке dist/

═══════════════════════════════════════════════════════════════

ТЕХНОЛОГИИ:
- Node.js
- Puppeteer (HTML → векторный PDF)
- pdf-lib (объединение страниц PDF)

═══════════════════════════════════════════════════════════════

ТЕКУЩЕЕ СОСТОЯНИЕ:
✓ HTML файлы созданы вручную
✓ Векторные PDF скрипты работают
✓ Векторные PDF файлы сгенерированы

ВОЗМОЖНЫЕ УЛУЧШЕНИЯ:

СТИЛЬ И ТРЕБОВАНИЯ К КАРТОЧКАМ:

СТРУКТУРА КАРТОЧЕК:
1. Первая карточка (титульная/лицевая):
   - Класс: .card-title
   - Размер: 600px × 800px
   - Фон: черный с градиентным свечением (фиолетовые оттенки)
   - Текст: белый, крупный (46px), жирный (900), uppercase
   - Акцент: желтый цвет (#FFE66D) для ключевых слов
   - Расположение: строки 178-183 в HTML файлах

2. Обычные карточки:
   - Класс: .card
   - Размер: 600px ширина, padding 50px 60px
   - Фон: белый
   - Шрифты: заголовки 30px/24px, текст 20px

ТРЕБОВАНИЯ К ТАБЛИЦАМ:
✓ Используем таблицы для структурированных данных
✓ Стиль: border с пунктирной линией, скругленные углы
✓ Фон таблицы: полностью залит цветом (не только заголовки)
✓ Заголовки (th): более темный оттенок для акцента
✓ Ячейки (td): светлый оттенок фона
✓ НЕ использовать нумерацию в первой колонке (если не требуется явно)
✓ Разделять данные на логические колонки (2-3 колонки оптимально)
✓ Примеры правильных таблиц:
   - Действие | Описание
   - Услуга | Когда нужна | Цена
   - Поле | Как заполнить
   - Требование | Что делать
